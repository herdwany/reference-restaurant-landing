# Backup Policy

## الجدولة

- تصدير شهري لكل `restaurant` بصيغة JSON.
- تصدير CSV للطلبات والحجوزات.
- الاحتفاظ بآخر 3 نسخ.
- نسخ صور العميل المهمة من `restaurant-assets`.

## قبل التعديلات الكبيرة

- export لبيانات المطعم.
- git commit للكود.
- توثيق التغيير المطلوب والوقت والمسؤول.

## الاستعادة

خطوات الاستعادة يجب أن تشمل:

1. إنشاء نسخة Database أو Table مؤقتة عند الحاجة.
2. استيراد بيانات restaurant.
3. استيراد dishes/offers/settings/content.
4. استيراد orders/reservations إذا كانت ضمن نطاق الاستعادة.
5. التحقق من `restaurantId`.
6. اختبار الموقع ولوحة التحكم.

## Audit Logs

يجب كتابة `audit_logs` لأي عملية:

- حذف.
- تعديل إعدادات حساسة.
- تغيير صلاحيات.
- تغيير حالة طلب أو حجز.
- عملية export أو restore.
